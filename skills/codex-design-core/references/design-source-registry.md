# Design Source Registry for Research

Use the sources below to support visual research. Extract principles and patterns; do not copy source material.

## iOS and mobile app UI

- Mobbin: search latest iOS apps by category, onboarding, paywall, dashboard, profile, editor, checkout, AI assistant, search, and settings.
- 60fps Design: use for high-polish motion, mobile interaction, and app interface references.
- Pinterest mobile boards: use for broad mood and graphic direction; verify with stronger product references before implementing.

## Web and landing pages

- Landing Love: use for landing-page sections, pricing, hero, proof, and conversion patterns.
- Godly Website: use for high-end web art direction and interaction benchmarks.
- Dribbble web/product shots: use for layout exploration and visual mood only.

## Graphic and brand direction

- Pinterest graphic design boards: use for composition, type treatments, editorial systems, and campaign mood.
- Dribbble product design and animation: use for interaction and visual language exploration.

## Component and pattern implementation

- shadcn/ui documentation and registry.
- coss/ui Empty for empty states.
- Fluid Functionalism for surface/context elevation.
- Liquid Metal and Border Beam for user-requested premium accent effects.
- Progressive Blur for scroll and overlay refinement.
- Spell for shadcn ecosystem component inspiration.

## Research note format

```xml
<design_research>
  <category>Product category and platform.</category>
  <sources>Sources inspected.</sources>
  <patterns>Reusable patterns observed.</patterns>
  <anti_patterns>What to avoid.</anti_patterns>
  <translation>How this becomes original UI for this product.</translation>
</design_research>
```
