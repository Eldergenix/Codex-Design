# Visual Inspiration Research

Use this skill to gather pattern-level design inspiration without copying proprietary work.

## Research rules

- Search by product category, platform, and UI pattern.
- Prefer official component docs for implementation APIs.
- Use visual inspiration for mood, hierarchy, composition, density, motion, and interaction patterns.
- Do not clone distinctive layouts, illustrations, brand systems, screenshots, command structures, or proprietary workflows.
- Capture insights in your own words.

## Source strategy

For mobile:

- Mobbin for real app flows.
- 60fps Design for motion and app polish.
- Pinterest mobile app boards for broader aesthetic mood.

For web:

- Landing Love for landing patterns.
- Godly Website for high-end creative direction.
- Dribbble web/product shots for visual exploration.

For components:

- shadcn/ui and coss/ui docs for source-owned component APIs.
- Fluid Functionalism for surface/elevation model.
- Liquid Metal, Border Beam, and Progressive Blur only when premium accents are requested.

## Stop rule

Stop researching once there is enough evidence to define direction, component anatomy, interaction expectations, and implementation constraints. Do not browse indefinitely.
